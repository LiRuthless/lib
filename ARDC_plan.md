# ARDC 报告

## 第一节：现有PI速度环配置

### 1.1 速度环PI参数

当前代码中速度环采用 **PI 控制器**（微分系数 KD_v 始终为 0，未启用 D 项）。参数在 `project/user/main.c` 中硬编码初始化，同时支持通过菜单或无线串口在线调参。

| 参数 | 符号 | 当前值 | 单位/说明 | 历史注释值 |
|------|------|--------|-----------|------------|
| 比例系数 | **KP_v** | **3.46** | 无量纲 | 4.35 |
| 积分系数 | **KI_v** | **0.39** | 无量纲 | 0.10 |
| 微分系数 | KD_v | 0.00 | 未使用 | — |

- **初始化位置**：`project/user/main.c` 第 28~29 行
- **在线调参入口**：`project/code/menu.c`（屏幕菜单）/ `project/code/config.c:uart_adjust()`（无线串口指令 `vp`/`vi`）
- **参数存储**：`menu.c` 中使用 `PidParams` 结构体在 RAM 中做一份备份，支持一键还原

### 1.2 控制器实现形式

代码同时实现了 **增量式 PID** 与 **位置式 PID**，通过宏 `USE_POSITION_PID`（`project/code/motor.h:21`）切换：

```c
#define USE_POSITION_PID  (1)   // 1=位置式, 0=增量式
```

当前选用的是 **位置式速度 PID**，对应函数为 `PID_L_pos()` 与 `PID_R_pos()`（`project/code/pid.c` 第 133~241 行）。位置式实现的核心公式：

```
P_out = KP_v * error
I_out += KI_v * error       // 积分累积
PID_base = P_out + I_out
```

随后叠加 **死区前馈**（Dead-zone Feedforward）：

```
if (target > 0)  PID_out = PID_base + MOTOR_DEAD_ZONE
if (target < 0)  PID_out = PID_base - MOTOR_DEAD_ZONE
if (target == 0) PID_out = PID_base       // 确保能真正停车
```

其中 `MOTOR_DEAD_ZONE = 1250`（`project/code/pid.h:23`），用于抵消直流电机的静摩擦死区。

**关键保护机制**：
- **输出限幅**：`±MAX_SPD_OUT = ±6000`
- **积分限幅**：`±(MAX_SPD_OUT - MOTOR_DEAD_ZONE) = ±4750`，防止积分暗饱和
- **方向切换清积分**：当目标速度由正变负（或反之）时，自动清零积分项，避免反向拖动

### 1.3 控制周期分析

#### 1.3.1 周期配置

速度环由 **TIM4 周期性中断定时器（PIT）** 驱动，控制周期为：

| 项目 | 配置 |
|------|------|
| 定时器通道 | TIM4_PIT（AI8051U 的 Timer4 周期性中断） |
| 初始化函数 | `pit_ms_init(PIT_SP, 10)` |
| 中断周期 | **10 ms** |
| 控制频率 | **100 Hz** |

相关代码分布：
- 宏定义：`project/code/config.h:16`
- 周期初始化：`project/code/config.c:40`
- 中断绑定：`project/user/main.c:22`
- 中断服务函数：`project/user/main.c` 的 `pit_speed()`

#### 1.3.2 中断服务流程

```
TIM4 中断 (每 10 ms)
    │
    ├─→ CLR_WDT = 1          // 喂看门狗
    ├─→ read_encoder()       // 读取编码器脉冲 → 低通滤波得到 real_speed
    └─→ motor_control()      // 调用 PID_L_pos() / PID_R_pos() → 输出 PWM
```

- **编码器读取**：`read_encoder()`（`project/code/motor.c:72`）读取左右编码器脉冲，经一阶低通滤波（滤波系数 `alpha = 0.65`）后得到 `real_speed_L` / `real_speed_R`，随后清零编码器计数，保证每次 10 ms 窗口内采集的是**增量脉冲**。
- **电机执行**：`motor_control()`（`project/code/motor.c:28`）根据 PID 输出设置 DIR 方向引脚与 PWM 占空比。

#### 1.3.3 与芯片时钟的关系

- **系统主频**：`clock_init(SYSTEM_CLOCK_40M)` → **40 MHz**
- **PWM 频率**：电机 PWM 初始化时设置为 **17 kHz**（`project/code/motor.c:119`）
- **定时器分辨率**：逐飞库中 `pit_ms_init` 宏定义为 `pit_init(pit_n, ms * system_clock / 1000)`。对于 40 MHz 主频、10 ms 周期，`pit_init` 得到的 `period = 400000`。该值小于 `255×65535`，因此 Timer4 自动配置为 **1T 模式**（不分频，指令周期 = 25 ns），预分频 `TM4PS = 12`，重装载值约 `30769`，实际计时精度由 16 位计数器决定，误差在纳秒级，远高于控制周期需求。

**Timer4 可配置的最小周期**：
- 1T 模式下，16 位定时器 + 8 位预分频，可覆盖 `1 ~ 16711680` 个时钟周期。
- 以 40 MHz 计算，理论最短中断周期约为 **1 µs**（`period = 40`），但受限于中断上下文保存/恢复及用户代码执行时间。
- `pit_speed()` 中仅执行：喂狗（1 条指令）+ 读编码器（约 20~30 条）+ 位置式 PID（浮点乘加，约 100~200 条）+ PWM 设置（约 10 条），总指令数 **< 300 条**。
- 在 40 MHz 1T 模式下，300 条指令约 **7.5 µs**。即便考虑中断开销和浮点模拟，整个 ISR 耗时通常 **< 50 µs**。
- 这意味着中断占用率仅约 `50 µs / 10 ms = 0.5 %`，CPU 几乎空转。**把周期缩短到 1 ms（占用率 5 %）甚至 0.5 ms（占用率 10 %）都完全有余量**。

#### 1.3.4 周期评估

| 指标 | 当前值 | 说明 |
|------|--------|------|
| 控制周期 T | 10 ms | 100 Hz |
| PWM 频率 | 17 kHz | 约为控制周期的 170 倍，电流环足够细腻 |
| 编码器采样 | 10 ms | 与 PID 同频，无额外采样延迟 |

**潜在瓶颈**：
1. **100 Hz 控制周期是带宽瓶颈**：10 ms 采样周期对应的奈奎斯特频率仅 50 Hz。考虑零阶保持（ZOH）效应后，速度环可稳定达到的控制带宽通常不超过 `f_s/10 ~ f_s/20`，即 **5 ~ 10 Hz**。若电机机械时间常数较小（例如 < 50 ms），100 Hz 的采样率会成为相位裕度的主要限制因素。

2. **编码器低通滤波器带宽尚可**：`alpha = 0.65` 的一阶 IIR 滤波器 `y[n] = α·x[n] + (1-α)·y[n-1]` 对应的后向欧拉离散时间常数为
   ```
   τ = T·(1-α)/α = 0.01 × 0.35 / 0.65 ≈ 5.4 ms
   ```
   -3 dB 截止频率约为
   ```
   f_c = 1/(2πτ) ≈ 29.5 Hz
   ```
   滤波器本身并不慢（29.5 Hz 远高于速度环控制带宽），等效群延迟在低频处约为 `τ ≈ 5.4 ms`，相当于约 **0.5 个控制周期**。真正限制系统响应速度的并非滤波器，而是 **100 Hz 的采样率**。

3. **PWM 与电流环足够快**：17 kHz PWM 频率约为控制周期的 170 倍，电流/力矩建立远快于速度环，不构成瓶颈。

### 1.4 为什么 100 Hz 会限制带宽——定量分析

很多智能车代码把速度环放在 10 ms（100 Hz），觉得这“够用”。但从控制理论看，**采样率直接封顶了速度环可达到的带宽**。

#### 1.4.1 采样系统的带宽上限

离散控制系统中，零阶保持器（ZOH）会引入额外相位滞后：

```
φ_ZOH(ω) = -ωT/2
```

在穿越频率 `ω_c`（即开环增益为 1 的频率）处，ZOH 造成的相位损失约为

```
Δφ ≈ -ω_c · T / 2 × (180/π)  [度]
```

若要求系统有 **45° 相位裕度**，ZOH 最多只能贡献约 **15°** 的相位损失（其余 30° 留给 PI 控制器和被控对象）。于是：

```
ω_c · T / 2 ≤ 15° × π/180 ≈ 0.262 rad
ω_c ≤ 0.524 / T = 0.524 / 0.01 = 52.4 rad/s
f_c = ω_c / (2π) ≈ 8.3 Hz
```

也就是说，**在 10 ms 采样周期下，速度环的稳定控制带宽很难超过 8 Hz 左右**。如果你的电机机械时间常数是 20~30 ms（对应带宽 5~8 Hz），这个采样率刚好够；但如果电机响应更快，或者你需要更快的速度跟踪（例如急加减速、坡道补偿），100 Hz 就会成为硬瓶颈。

#### 1.4.2 当前 PI 参数在离散域的含义

现有参数 Kp = 3.46、Ki = 0.39 是在 **10 ms 周期** 下整定的。若把周期缩短到 `T_new`，必须同步缩放积分系数，否则积分作用会变弱或变强：

```
Ki_new = Ki_old × (T_old / T_new)
```

例如：
- 若改为 **5 ms（200 Hz）**：`Ki_new = 0.39 × 2 = 0.78`
- 若改为 **2 ms（500 Hz）**：`Ki_new = 0.39 × 5 = 1.95`

比例系数 Kp 理论上不随周期改变（因为它是瞬态增益），但实际操作中由于量化、噪声和高频未建模动态，缩短周期后可能需要微调 Kp。

#### 1.4.3 提升到更高频率的改动量

以当前代码为基础，把速度环从 10 ms 提升到 **5 ms** 或 **2 ms**，只需改一行代码：

```c
// project/code/config.c:40
// pit_ms_init(PIT_SP, 10);   // 原 100 Hz
pit_ms_init(PIT_SP, 5);       // 改为 200 Hz
// 或
pit_ms_init(PIT_SP, 2);       // 改为 500 Hz
```

同时注意：
- `speed_test2()` 和 `speed_test()` 中的 `cnt >= 80` 计数器是基于 10 ms 假设的（`80 × 10 ms = 800 ms`）。若改为 5 ms，应改为 `cnt >= 160`；若改为 2 ms，应改为 `cnt >= 400`。
- `main.c` 中的串口打印 `system_delay_ms(100)` 不受速度环周期影响，可保留。
- 看门狗喂狗周期（约 140 ms）远大于 2 ms，不受影响。

**结论**：代码本身没有任何结构性障碍阻止你把速度环跑到 200 Hz 甚至 500 Hz。真正限制带宽的，不是 AI8051U 的算力，也不是滤波器，而是**当前那行 `pit_ms_init(PIT_SP, 10)`**。

### 1.5 小结

现有速度环配置可归纳为：

- **PI 参数**：Kp = 3.46，Ki = 0.39，无 D 项；
- **实现方式**：位置式 PID + 死区前馈 + 积分限幅 + 方向切换清积分；
- **控制周期**：10 ms（100 Hz），由 TIM4_PIT 中断驱动；
- **硬件基础**：AI8051U @ 40 MHz，电机 PWM 17 kHz，编码器 10 ms 增量采样；
- **带宽瓶颈**：100 Hz 采样率引入的 ZOH 相位滞后把速度环稳定带宽限制在约 **8 Hz** 以内；滤波器截止频率约 29.5 Hz，不构成瓶颈；CPU 中断负载仅 ~0.5 %，完全有能力跑到 200~500 Hz。

> **注**：以下各节将在此基础上，针对控制周期优化、参数自整定或前馈补偿等方向展开分析与实验。

## 第二节：一阶ADRC速度环参数设计方案

### 2.1 ADRC 基本结构

采用**一阶线性自抗扰控制（LADRC）**，将被控对象（电机速度环）统一建模为：

```
ẏ = b·u + f
```

- **y**：编码器反馈速度，单位 `pulse/10ms`
- **u**：电机 PWM 控制量，范围 `[-6000, 6000]`
- **b**：控制增益（真实值未知）
- **f**：**总扰动**，包含：内部参数摄动、负载转矩变化、摩擦非线性、电池压降、坡道重力分量、未建模动态等

ADRC 的核心思想是：用**扩展状态观测器（ESO）**实时估计总扰动 `f̂`，然后在控制律中将其补偿掉，把原系统还原为纯积分器 `ẏ ≈ b·u₀`，最后用简单的比例控制实现期望动态。

### 2.2 步骤一：确定 b₀（控制增益估计）

b₀ 是真实控制增益 b 的估计值，是 ADRC 设计中**唯一需要知道的被控对象参数**。ADRC 对 b₀ 的不精确性具有天然鲁棒性——只要数量级正确（0.5~2 倍真实值），ESO 的扰动估计会自动吸收剩余误差。

#### 2.2.1 推荐方法：开环阶跃标定

这是最准确的方法，建议上线前在实验台上完成：

1. **固定 PWM 输出**：令 `u₀ = 2000`（避开死区 1250，约为满幅的 1/3）
2. **记录速度响应**：通过串口输出 `real_speed` 曲线，或观察屏幕上的速度值
3. **估计方法（二选一）**：
   - **初始斜率法**：`b₀ = Δy/Δt / u₀`，取响应起始段的平均斜率
   - **稳态增益法**：测出稳态速度 `y_ss` 和达到 63% 稳态值的时间常数 `τ`，则 `b₀ = y_ss / (τ · u₀)`

> **示例**：若 u₀=2000 时稳态速度 y_ss=1200 pulse/10ms，τ=60ms，则 `b₀ = 1200/(0.06×2000) = 10`。

#### 2.2.2 理论估算法（无实验条件时）

智能车常用直流减速电机的典型参数范围：

| 参数 | 典型范围 | 说明 |
|------|---------|------|
| 稳态增益 K | 0.3 ~ 0.8 | (pulse/10ms)/PWM，满幅 6000 对应几百~几千 pulse/10ms |
| 机械时间常数 τ | 30 ~ 100 ms | 含减速箱惯量 |
| 控制增益 b = K/τ | **3 ~ 27** | (pulse/10ms)/(PWM·s) |

取设计中心值：

```
b₀ = 8
```

**鲁棒工作范围**：`b₀ ∈ [2, 20]` 内均可获得良好性能。

#### 2.2.3 与现有 PI 参数的自洽验证

ADRC 在无扰动、且 b₀ 准确时，控制律退化为纯比例控制：

```
u = (ω_c / b₀) · e
```

其中 `ω_c` 是 ADRC 的控制器带宽（ rad/s ），`e = r - y`。

- 若取保守带宽 `ω_c = 31.4 rad/s`（5 Hz），则等效比例增益 `Kp_eq = 31.4/8 ≈ 3.9`，与现有 `KP_v = 3.46`**量级一致**。
- 若取激进带宽 `ω_c = 62.8 rad/s`（10 Hz），则 `Kp_eq = 62.8/8 ≈ 7.9`，对应比现有 PI 更快的响应。

这说明 **b₀ = 8 与现有 PI 参数在物理上是自洽的**，不需要大幅修正。

### 2.3 步骤二：确定观测器带宽 ω_o 与控制器带宽 ω_c

#### 2.3.1 带宽设计原则

| 原则 | 内容 |
|------|------|
| ω_c 上限 | 受采样频率约束：`ω_c < ω_s / 10`，即 `f_c < f_s / 20` |
| ω_o / ω_c 比值 | 通常取 **3 ~ 10**：ω_o 太小则扰动估计慢，太大则对噪声敏感 |
| ω_o 上限 | 受离散化稳定性约束：前向欧拉下 `ω_o < 2/T`；保守取 `ω_o < ω_s / 3` |

其中 `ω_s = 2π·f_s` 为采样角频率。

#### 2.3.2 方案A：保守方案（保持 100 Hz 采样）

| 参数 | 符号 | 值 | 说明 |
|------|------|-----|------|
| 采样周期 | T | 10 ms | 不改定时器 |
| 采样频率 | f_s | 100 Hz | |
| 控制器带宽 | ω_c | **31.4 rad/s** | 2π × 5 Hz，闭环带宽约 5 Hz |
| 观测器带宽 | ω_o | **94.2 rad/s** | 3 × ω_c，约 15 Hz |
| ESO 增益 β₁ | 2ω_o | 188.4 | 1/s |
| ESO 增益 β₂ | ω_o² | 8874 | 1/s² |
| 控制器增益 | k_p | 31.4 | 1/s，等于 ω_c |

**离散稳定性校验**（前向欧拉）：

连续 ESO 极点在 `s = -ω_o = -94.2`（二重）。离散化后特征值模约为 **0.058**（远小于 1），**非常稳定**。观测器收敛时间约 `3/ω_o ≈ 32 ms`（3 个控制周期）。

**适用场景**：不想改动定时器配置、希望平稳过渡、对噪声抑制要求高。

#### 2.3.3 方案B：激进方案（提升到 500 Hz 采样）

| 参数 | 符号 | 值 | 说明 |
|------|------|-----|------|
| 采样周期 | T | 2 ms | 改 `pit_ms_init(PIT_SP, 2)` |
| 采样频率 | f_s | 500 Hz | |
| 控制器带宽 | ω_c | **75.4 rad/s** | 2π × 12 Hz，闭环带宽约 12 Hz |
| 观测器带宽 | ω_o | **377 rad/s** | 5 × ω_c，约 60 Hz |
| ESO 增益 β₁ | 2ω_o | 754 | 1/s |
| ESO 增益 β₂ | ω_o² | 142129 | 1/s² |
| 控制器增益 | k_p | 75.4 | 1/s |

**离散稳定性校验**：离散 ESO 特征值模约为 **0.246**（< 1），**稳定**。观测器收敛时间约 `3/ω_o ≈ 8 ms`（4 个控制周期）。

**配套改动**：
- 编码器低通滤波 `alpha = 0.65` 的 29.5 Hz 截止频率会成为瓶颈，建议**取消或提高 alpha 至 0.85+**，让 ESO 自己承担滤波作用。
- `speed_test()` 和 `speed_test2()` 中的计数器需按比例缩放（`cnt >= 400` 对应 800 ms）。

**适用场景**：需要更快的速度跟踪（如高速循迹、急加减速、坡道补偿）。

### 2.4 离散实现公式

#### 2.4.1 扩展状态观测器（ESO）

```c
// 每个控制周期调用，T 为采样周期
e  = z1 - y;                              // 观测误差，单位 pulse/10ms

z1 = z1 + T * (z2 - beta1 * e + b0 * u);  // 状态估计更新
// z1 跟踪 y，单位 pulse/10ms

z2 = z2 + T * (-beta2 * e);               // 扰动估计更新
// z2 跟踪 f，单位 (pulse/10ms)/s
```

> **物理意义**：z₂ 实时估计了所有“让电机不按期望转”的因素——电池压降、负载变化、摩擦、坡道分量等。只要 z₂ 收敛快，系统就表现得像一个理想的积分器。

#### 2.4.2 控制律

```c
u0 = kp * (r - z1);                       // 线性反馈，单位 (pulse/10ms)/s
u  = (u0 - z2) / b0;                      // 扰动补偿 + 增益归一化，单位 PWM
```

#### 2.4.3 输出处理（沿用现有机制）

ADRC 计算出的 `u` 直接替换现有 PID 输出，后续处理完全沿用：

```c
// 死区前馈 + 限幅（与现有 motor_control 逻辑一致）
if      (target > 0)  u += MOTOR_DEAD_ZONE;
else if (target < 0)  u -= MOTOR_DEAD_ZONE;

if (u >  MAX_SPD_OUT) u =  MAX_SPD_OUT;
if (u < -MAX_SPD_OUT) u = -MAX_SPD_OUT;

// 设置方向 + PWM（与现有代码一致）
if (u >= 0) { DIR = 正转; pwm =  u; }
else        { DIR = 反转; pwm = -u; }
```

### 2.5 参数汇总与对比

| 参数 | 符号 | 方案A (100Hz) | 方案B (500Hz) | 单位 | 物理意义 |
|------|------|--------------|--------------|------|---------|
| 控制增益估计 | **b₀** | 8 | 8 | (pulse/10ms)/(PWM·s) | 唯一需标定的对象参数 |
| 控制器带宽 | ω_c | 31.4 | 75.4 | rad/s | 决定闭环响应速度 |

### 2.6 代码实现建议

1. **数据结构**：为左右轮各定义一个 ADRC 结构体
   ```c
   typedef struct {
       float z1;       // 状态估计（跟踪速度）
       float z2;       // 扰动估计
       float b0;       // 控制增益估计
       float beta1;    // ESO 增益1
       float beta2;    // ESO 增益2
       float kp;       // 控制器增益
   } ADRC_Controller;
   ```

2. **初始化位置**：在 `All_init()` 之后、主循环之前初始化
   ```c
   ADRC_Controller adrc_L = {0, 0, 8.0f, 188.4f, 8874.0f, 31.4f};
   ADRC_Controller adrc_R = {0, 0, 8.0f, 188.4f, 8874.0f, 31.4f};
   ```

3. **中断替换**：在 `pit_speed()` 中，用 ADRC 计算替换 `PID_L_pos()` / `PID_R_pos()` 调用
   ```c
   speed_outL = adrc_update(&adrc_L, target_speed_L, real_speed_L);
   speed_outR = adrc_update(&adrc_R, target_speed_R, real_speed_R);
   ```

4. **调参接口**：沿用现有 `KP_v`/`KI_v` 的调参菜单，映射关系：
   - `KP_v` 旋钮 → 映射为修改 `kp`（或等效修改 ω_c）
   - `KI_v` 旋钮 → 映射为修改 `b0`（影响稳态补偿效率）
   - 或者更直观：菜单直接显示 `ω_c` 和 `ω_o`，步进调整

5. **b₀ 在线标定**：新增一个调试模式（例如按键组合触发）
   - 固定 PWM = 2000，持续 200 ms
   - 记录速度初始斜率，自动计算并显示 b₀
   - 按 OK 写入当前 ADRC 实例

### 2.7 与 PI 的对比优势

| 对比项 | 现有 PI | 一阶 ADRC |
|--------|---------|-----------|
| 抗负载扰动 | 靠积分慢慢消除，有超调 | ESO 实时估计补偿，响应快 |
| 抗电池压降 | 积分windup，可能饱和 | 自动纳入总扰动 z₂，无windup |
| 坡道/摩擦补偿 | 需手动加前馈或调参 | 自动补偿，无需额外参数 |
| 参数整定 | Kp/Ki 耦合，调参困难 | ω_c 管响应、ω_o 管观测，物理意义清晰 |
| 模型依赖 | 无 | 仅需 b₀ 的数量级，鲁棒性强 |

### 2.8 小结

本方案基于现有 PI 速度环，设计了一阶线性 ADRC 的两套参数：

- **b₀ = 8**：通过典型电机参数估算，并与现有 Kp=3.46 自洽验证；建议上线前用开环阶跃法精确标定。
- **方案A（100Hz）**：ω_c = 31.4 rad/s，ω_o = 94.2 rad/s，改动最小，直接替换 PID 即可。
- **方案B（500Hz）**：ω_c = 75.4 rad/s，ω_o = 377 rad/s，需改定时器周期并调整编码器滤波，但带宽提升 2.4 倍。

下一步可先在方案A基础上实现 ADRC，与现有 PI 做 A/B 对比测试，验证 b₀ 估计和扰动补偿效果后，再决定是否升级到方案B。
